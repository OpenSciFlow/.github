# Contributing to OpenSciFlow

OpenSciFlow is currently an early initiative. Contributions should be small, reviewable, and grounded in real tools or workflows.

## Good first contributions

- Add or correct a related project.
- Improve a plugin manifest example.
- Add license/citation metadata.
- Review a workflow template.
- Improve documentation or translations.

## Contribution rules

- Do not claim partnership with another project unless maintainers explicitly approve it.
- Link to upstream documentation and citations.
- Keep scientific claims bounded.
- Add limitations when proposing workflow outputs.
- For plugin manifests, include license, citation, validation, and safety notes.

## Review checklist

- Is the source link correct?
- Is the license clear?
- Is the citation clear?
- Are required inputs/outputs explicit?
- Is execution local/HPC-safe?
- Are limitations stated?
